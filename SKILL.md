# 清华大学研究生学位论文格式整改

把用户写好的论文 docx 按 2025 版《研究生学位论文写作指南》整改格式。三条铁律:

1. **内容零改动**——任何文字(包括标点、空格、编号文字)都不许增删改。脚本内置整改前后全文逐字比对,FAIL 即放弃保存。
2. **格式全合规**——以 `references/format-spec.md` 为唯一标准,数值不靠记忆。
3. **修改全记录**——每一处修改记录"位置 + 属性 + 旧值 → 新值",最终生成用户可核对的修改清单。

## 依赖

`pip install python-docx lxml`(若环境无网络,先确认已预装)。
输入必须是 .docx;若用户给 .doc,先转换:
`soffice --headless --convert-to docx 文件.doc --outdir 目录/`

## 工作流(先识别学位类型,再七步整改,全自动,顺序执行)

本 skill 适用于**研究生学位论文(博士 / 硕士)**:博士与硕士的正文格式规范完全相同,差异只在封面与学位标识,因此第 1~7 步整改流程对博硕完全一致,只需先用第 0 步识别学位类型供核对。

### 第 0 步:识别学位类型(博士 / 硕士)

```bash
python scripts/detect_degree.py 论文.docx -o degree_info.json
```

读取封面与前置页,依据中文封面学位类别行("……博士学位论文" / "……硕士学位论文",专业学位为"……博士/硕士专业学位论文")与英文封面学位行("Doctor of Philosophy" / "Master of …"、"Dissertation submitted to …" / "Thesis submitted to …")判定 `degree_level`(博士/硕士)与 `degree_category`(学术/专业),连同证据与置信度写入 degree_info.json。
判定只用于**核对封面与交付说明**,不影响第 1~7 步整改(博硕正文规范相同)。当 `confidence` 为 low/none(常见于封面单独成文件、未随正文上传)时,主动向用户确认学位类型后再继续。
同时从封面"申请人/研究生/作者"栏抓取作者姓名,写入 `author_name`,并给出**建议输出文件名** `suggested_filename`(=「姓名+博士/硕士论文+修改后版本」,如 `潘俊生硕士论文修改后版本`)。第 7 步交付时用它命名最终 docx;若 `author_name` 为空(没抓到),按个人简历页信息或向用户确认后命名。

### 第 1 步:解析分类

```bash
python scripts/classify.py 论文.docx -o structure_map.json
```

输出 `paragraphs`(逐段角色)、`tables`(表格清单:数据表 three_line / 图片排版表 image_layout)、`numbering_census`(编号体例"-"/"."统计)。
章节标题既认"第N章 / N.M"这类**文字编号**,也认**Word 自动编号 / 标题1-标题4 样式**(大纲级别)——标题文字里没有"第N章"、靠自动编号的章节也能识别,省去大量人工修正。目录行(行尾带页码,含制表符对齐)不会被误判成章标题。

### 第 2 步:复核图谱(由 Claude 完成,不需用户动手)

classify.py 是启发式的,必然有误判。Claude 通读 structure_map.json 并修正:

- 各部分边界:摘要、目录、正文第 1 章、参考文献、附录、致谢的起止段。
- `equation`:无编号公式段、图片公式易被判成 body。
- `fig_caption` / `table_caption`:英文对照题注(Fig 2.1 …)、无空格编号的题注。
- `tables`:**封面/前置区(摘要之前)的所有表格自动标 `skip` 完全不动**(模板封面就是用表格排版的:题目、培养单位/作者/导师信息表、英文封面、答辩委员会名单——一旦画三线、把字号压成 11pt 就会把封面搞崩)。正文里含图的占位/布局表判为 `image_layout`(清边框),数据表为 `three_line`(画三线)。
- `symbol_entry`:符号和缩略语说明区已自动识别并整段保留版式(tab 两列对齐/悬挂缩进不动),核对边界即可。
- `cv_entry`:"个人简历、在学期间完成的相关学术成果"整节也自动保留模板版式(居中子标题、"学术论文/专利/参与科研项目"类别标签、参考文献式成果条目混排,均不动),核对边界即可。
- 章节标题既认"第N章/N.M"文字编号,也认 **Word 自动编号 / 标题1-标题4 样式**(大纲级别);正文里以"第N章…"开头的**长句**(>40字或句末带句号)不会被误判成章标题。
- 摘要之前的段一律 `cover`;**封面/前置区分节的页边距(中文封面 6/6/4/4、书脊、英文封面 5.5/5/3.6/3.6 等)整改时自动保留,不会被改成正文的 3/3/3/3**。

修正直接改 JSON 的 `role` 字段。结构特殊(著者-出版年制、人文社科脚注体系)时向用户确认。

### 第 3 步:检查(dry-run)

```bash
python scripts/apply.py 论文.docx -m structure_map.json --dry-run -r before_report.json
```

把"整改前共 N 处不合规"概况告诉用户,然后继续。

### 第 4 步:整改字体/段落/表格

```bash
python scripts/apply.py 论文.docx -m structure_map.json -o step4.docx -r format_report.json
```

覆盖:页边距、各角色的中西文字体/字号/对齐/行距/段前后/缩进、表格单元格文字、**三线表线型(上下 1.5 磅、表头下 1 磅、清内线)**;`image_layout` 图片占位表**清除全部边框**(不动其单元格内容),`symbol_entry` 符号说明区**整段不动**(保留专用 tab 对齐)。
结尾必须出现 `内容完整性校验: PASS`,FAIL 时脚本拒绝保存。

### 第 5 步:分节、篇眉、页码(全自动)

```bash
python scripts/sections.py step4.docx -m structure_map.json -o step5.docx -r sections_report.json
```

自动完成:每部分前插分节符(正文第一章为**奇数页**分节符,实现"另页右页开始");每节篇眉=该部分章标题(五号宋体/TNR 居中,奇偶相同、首页同),**篇眉文字下方加 0.75 磅细横线**(封面等空篇眉不画线);页脚 PAGE 域居中 TNR 五号;**摘要起罗马数字Ⅰ连续编排,正文第 1 章起阿拉伯数字 1 连续编排**;摘要前的封面区篇眉页脚清空;写入 updateFields——**Word 打开文件时自动刷新目录/清单页码,用户无需按 F9**。
页眉页脚采用"整体清空容器(含模板自带的 `w:sdt` 页码内容控件)再写入"的方式,避免与库页码叠加出现**两个页号**;前置区分节数按 `总节数−部分数` 反算,兼容封面区天然多分节(中文封面/书脊/英文封面各自独立)。
**单面打印格式**:封面/英文封面/名单/授权说明这些前置页与摘要都自动置于**奇数页(另页右页)**——双面打印时奇数页分节符会在每页后自动补一张空白页(原本已是奇数页/已有空白页的不重复补;只对"有内容"的封面分节生效,文档里原有的空白分节保持原样)。用 `--no-front-blank` 可关闭补空白页。
若文档含封面页,用 `--cn-cover 部分序号` / `--en-cover 部分序号` 自动应用封面专用页边距。

### 第 6 步:编号体例统一(图/表 与 公式 各自独立)

**图/表 与 公式 是两套独立的编号序列**(通行约定:图表用 ".",公式用 "-"),分别统计、分别统一,**只在某一套自身混用时才改它**;两套各自一致时本步 0 改动:

```bash
python scripts/normalize_numbering.py step5.docx --auto -o step6.docx -r numbering_changes.json
```

`--auto`:每套有多数派取多数派,平局/空集回退默认(图表→".",公式→"-")。需要显式指定可用 `--figtab dot|hyphen` / `--formula dot|hyphen`;`--census` 只统计并分别标 `[一致]` / `[自身混用!]`;`--unify-all` 为旧版兼容(把两套强行统一,一般不用)。
这是流水线中**唯一会改文字的脚本**,改动严格限定为编号连接符一个字符,每处替换逐条记录。报告含 `target_figtab` / `target_formula` / `census_before:{figtab,formula}`;如有 `unresolved`(编号被拆进多个 run),Claude 用 python-docx 定位该段手工合并替换并补记录。

### 第 7 步:终验、清单、交付

```bash
python scripts/verify.py 论文.docx 最终.docx --allow-numbering
python scripts/report.py format_report.json -o 修改清单.md
```

verify.py 对比原始与最终文档全文:除已授权的编号连接符外,任何文字差异都 FAIL。
**最终 docx 用第 0 步 degree_info.json 里的 `suggested_filename` 命名**(=「姓名+博士/硕士论文+修改后版本.docx」,如 `潘俊生硕士论文修改后版本.docx`);抓不到姓名时再向用户确认。
交付内容(present_files):最终 docx + 修改清单.md(合并 format/sections/numbering 三份报告的要点)。
建议的自动化质检:`soffice --headless --convert-to pdf 最终.docx`,渲染 PDF 后抽查篇眉、页码、分节是否正确(容器内缺中文字体时文字可能显示异常,但版式检查仍有效)。

## 仍需告知用户的事项(不是操作,只是知会)

1. 封面、符号和缩略语说明、个人简历及学术成果等**学校模板专用版式整段保留不动**(含封面专用页边距与信息表),本 skill 只整改正文该改的部分,不生成也不重排这些模板内容。
2. 双面打印是打印机设置;文档侧的奇偶分页已由奇数页分节符保证。
3. 目录若是手打文本而非 Word 域:格式已统一、updateFields 不会作用于它,建议改用自动目录(可主动提出帮用户把手打目录替换为 TOC 域,经确认后执行)。

## 与规范有出入时的决策原则

- 拿不准某段角色:宁可标 `skip` 不动它并报告,不要猜。
- 规范允许二选一的(公式居中 vs 缩进两字符;编号 "-" vs "."):跟随论文现状中占多数的一种,并在清单中注明。
- 院系另有要求的:指南明确"以院系要求为准",用户提供院系规定时,院系规定优先,差异处在清单中标注。
- 用户论文是英文论文或中英混排特殊情况:先和用户确认适用范围再动手。

## 参考文件

- `references/format-spec.md` — 全部格式参数(字体/字号/行距/磅值/页面),动手前必读,数值以它为准。
- `references/document-structure.md` — 论文组成部分顺序与各部分要点。
- `scripts/spec.py` — format-spec.md 的代码化常量,二者必须一致。
- `scripts/detect_degree.py` — 识别学位类型(博士/硕士、学术/专业学位),输出 degree_info.json(博硕正文规范相同,仅用于核对封面与学位标识)。
